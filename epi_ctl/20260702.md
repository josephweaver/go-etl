# Epistemic Control Review: 2026-07-02

## Completed Slice

- Controller-startup-resolution 011 resolves and validates controller-owned
  filesystem paths after database startup and before later service construction.

## Concepts Introduced

- A narrow Go struct can carry related constructor inputs without becoming a
  second configuration model.
- An unqualified variable reference preserves the resolver's configured source
  precedence, including startup overrides.
- `filepath.IsAbs`, `filepath.Join`, and `filepath.Clean` perform lexical path
  normalization without accessing the filesystem.
- Injecting the working directory makes path resolution deterministic in tests.

## Review Questions

2. Why does the helper require an absolute working directory?
3. What guarantees does `filepath.Clean` provide, and what does it not check?
4. Why may the returned path struct be retained while the resolver should not be?
