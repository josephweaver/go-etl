5. From memory, sketch the variable precedence order from lowest to highest.

### Round 2

1. What changed about `internal/variable.Type` in the last slice? Why was a plain string type no longer enough?

## Observed Answers

The human correctly described the controller as the current owner of the work queue, including assignment, success/failure reporting, status, and raw work submission. The human also correctly distinguished the intended future workflow submission and compilation role from what is currently implemented.

The worker success path was traced accurately at a high level: worker requests work, controller moves pending to assigned, worker runs, worker reports success, worker asks for more work. One detail was imprecise: completed work is not currently moved into a completed collection. It is removed from `assigned`.

Failure prediction was strong. The human identified that abandoned work has no current protocol, that assigned work can become trapped if the worker dies or cannot report completion, and that a future timeout or caretaker mechanism will be needed.

The `OutputFilename` invariant was not understood. The human associated it with demo work but did not identify that it prevents path traversal and keeps work-item output inside configured mounted directories.

Variable precedence reconstruction was initially incomplete. The human missed `controller_env` and `worker_env`, and placed backend too early. After prompting, the human correctly supplied `controller_env` and `worker_env`.

The human understood the recent `Type` change at an operational level: moving away from a plain string toward a `Kind`-based representation so `list[T]` can be modeled. The answer did not mention the exact `Element *Type` structure, but it captured the reason for the change.

The human correctly remembered that `ParseLiteral` does not parse `object` or `list[T]` yet and that explicit rejection behavior was added.

The human correctly explained why fan-out should wait: fan-out depends on list values and on retrieving list data from structured objects through an accessor/query mechanism.
